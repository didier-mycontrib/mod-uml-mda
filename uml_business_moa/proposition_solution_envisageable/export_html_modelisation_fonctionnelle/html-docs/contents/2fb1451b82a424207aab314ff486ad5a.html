configurer les variantes d'un produit

UMLUseCase
modélisationFonctionnelle_artisanat_decoration :: 1_Modele_fonctionnel_siteArtisanat_deco :: configurer les variantes d'un produit

Description

précondition:

réalisationType sélectionnée et produit associé prêt à être configuré

scénario nominal:

  1. lister les variantes principales et complémentaires d'une réalisationType
  2. le client sélectionne une variante principale (enregistrée dans la config du produit)
  3. le client sélectionne d'éventuelles variantes complémentaires (ex: couleur)
  4. les variantes complémentaires sélectionnées par le client sont enregistrées dans la configuration du produit
  5. le système vérifie la suffisance et la cohérence des variantes sélectionnées
  6. le système affiche un résumé du produit configuré (prix , variantes choisies, ...)
