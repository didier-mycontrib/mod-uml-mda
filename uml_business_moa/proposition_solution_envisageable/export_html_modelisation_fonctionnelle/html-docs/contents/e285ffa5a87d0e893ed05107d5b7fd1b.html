passer_commande

UMLUseCase
modélisationFonctionnelle_artisanat_decoration :: 1_Modele_fonctionnel_siteArtisanat_deco :: passer_commande

Description

scénario nominal: 1) le système créer une ébauche de commande (etat=EN_ELABORATION) faisant office de caddy 2) le client sélectionne (si pas déjà fait) une des réalisations types 3) le système construit un début produit de produit sur mesure qui sera ultérieurement intégré dans une des lignes de commande 4) le client sélectionne les variantes d'une réalisation type pour configurer sont produit (voir sous U.C.) 5) le client choisi la quantité à commander pour le produit configuré 6) le client s'identifie et précise ses coordonnées (via une gestion de compte client) (voir sous U.C.) 7) le client valide sa commande 8) le client effectue le paiement en ligne (voir sous U.C.) 9) le système affiche un résumé des opérations effectuées
